IBD · Client Contact Repository · Build guide

One search, every contact

A view-only canvas app over one SharePoint list: type a RIM or borrower name, get the borrower's facility snapshot and everyone on the relationship — the BMO coverage team and the borrower's office contacts — straight from the Master List of Exposures. The app reads, never writes. Every step is self-contained, and everything — names, formulas, colors, sizes, spacing — has a copy button.

Read this first

What you're buildingSingle screen, master-detail: a search panel on the left, and on the right a borrower card (facility snapshot) plus two contact blocks — the BMO coverage team (RSMs, RM, RIM officer) and the borrower's office contacts (email addresses). Everything comes straight off the selected row of the Master List of Exposures. No Power Automate, no data entry, no other lists: the BMO team keeps maintaining the Master List exactly as they do today, and the app just makes it searchable.

Where did the four-list design go?The v3 spec designed three more lists (Borrower Contact Directory, Communication Purpose Master, Contact Communication Matrix) for per-purpose external contacts. Tap any chip to copyDark boxes have a Copy button. Every inline chip like galBorrowers — including every size, color and spacing value in the property tables — is tap-to-copy; the ⧉ mark is the reminder. Paste, never retype.
The visual system — five values you'll reuse everywhere

The app stays uncluttered because it uses one accent color, one grey, a fixed type scale and a fixed spacing scale. Copy these once, recognize them everywhere:

  • Navy accent (header, block headings, email links, highlight): RGBA(31, 59, 110, 1)
  • Body text (names, values): RGBA(27, 36, 54, 1)
  • Secondary grey (meta lines, hints, placeholders): RGBA(91, 102, 120, 1)
  • Card / panel background: RGBA(244, 245, 248, 1) — page behind panels: RGBA(237, 239, 244, 1)
  • Type scale: 20 (borrower name) · 15 (block headings) · 14 (facility line, emails) · 13 (contact rows) · 12 (meta, hints). Spacing scale: 4 (inside a pair) · 8 (between rows) · 12 (between blocks) · 16 (panel padding). Nothing else.

Font is Segoe UI (Power Apps' default — set it only if a control shows something else). If two things compete visually, the fix is always: smaller size, greyer color, or more gap — never a new color.

S5Selection powers the right panel

One line. Everything the right panel shows — facility figures, coverage team, office emails — is a column on the row the user just tapped, so remembering that row is the whole job.

galBorrowers · OnSelect

Set(varBorrower, ThisItem)
Why it's this simpleThe contact data lives on the Master List row — no second list to query, no collection to build, zero extra server calls per selection. The right panel just reads varBorrower.<column>.

Why .Value here but .DisplayName thereThe RSM columns are Lookups — they carry their text through .Value, and their extra fields arrive as separate read-only columns ('BMO RSM 1: Email Address', also a .Value). Rim Officer Name is a Person column — it carries the whole person record: .DisplayName, .Email. Confirmed from the live connector schema; IntelliSense will agree as you type.
A blank name making an awkward row?Hide that row the same way the email recipe does — e.g. lblRM · Visible → !IsBlank(varBorrower.RM). (The container already handles the no-selection case.)
Sizing sanity checkconCoverage's 216 = heading 24 + rows (22+20+22+20+22+22+20) + seven 4px gaps + 16px padding. If you add or remove a row, adjust the height by that row + 4.

S11Test everything

F5 from scrMain and run all ten. All must pass before publishing:

  • Open with nothing typed → full borrower list left, placeholder line right.
  • Partial borrower name → list narrows as you type.
  • Partial RIM → matching borrowers appear.
  • Gibberish → "No borrowers found".
  • Select a borrower → card shows name, RIM/office/grade line and the facility figures.
  • Coverage block shows RSM 1, RSM 2, RM, RIM officer names from the list.
  • Tap any email → a pre-addressed mail draft opens.
  • A borrower with a blank email (or blank 2nd office email) → that row is hidden, no empty gap.
  • Switch borrowers quickly → the right panel always matches the selection, no stale values.
  • App checker (stethoscope) → no delegation warnings on galBorrowers.
Check before moving onTen for ten. Any miss → the step that built that piece has the formula to re-check.
S12Publish & share — the victory lap
  1. File → Save → Publish → Publish this version.
  2. Power Apps home → the app's ⋯ menu → Share → add the IBD users or their security group as User (not Co-owner).
  3. Hand the BMO team this guide plus your filled-in V1 checklist, so future maintainers know the app's exact column dependencies.
Sharing the app does not grant list accessUsers also need Read on the Master List of Exposures — grant it on the BMO site. The app is view-only, so Read is all they need; don't grant Edit.
