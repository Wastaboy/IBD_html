1 · Overview & architecture

You are building a view-only Power Apps canvas app that lets IBD staff find a borrower by RIM or name and instantly see every operational contact for that borrower, grouped by communication purpose (audit confirmations, legal notices, daily transaction advices, and so on).

The core design principle from the specification: the Master List of Exposures remains the single source of truth for borrower information. The app reads it, never writes it, and never duplicates borrower data anywhere else.

Master List of Exposuresexisting borrower master (read-only)
→
Contact Communication Matrixwho · for which borrower · for what purpose
←
Borrower Contact Directorypeople & their details
Communication Purpose Masterdrives the grouping
→
Power Apps front endsingle screen · search → detail

The app is a single screen: a search panel on the left, and on the right a borrower info card above the borrower's contacts grouped by purpose. Because the purpose grouping is driven by the Communication Purpose Master list rather than hard-coded, new purposes — and new Wholesale Banking units later — appear in the app with zero redesign.

This guide contains no Power Automate steps and no data-entry screens: contacts and matrix rows are maintained directly in SharePoint by the BMO team. The app is strictly for finding and viewing.

2 · Prerequisites & verify your columns

What you need before starting

  • A Power Apps license that allows SharePoint connections (standard Microsoft 365 is enough — no premium connectors are used).
  • Read access to the BMO SharePoint site: https://bbkonline.sharepoint.com/sites/BMO.
  • The four lists below already created on that site, with at least a few rows of real or sample data in each (you cannot verify the app against empty lists).
Do this step — do not skip it. Every formula in this guide uses the column names from the specification document. If your real SharePoint columns are named differently, the formulas will error until you substitute your names. Names marked with an verify chip throughout the guide are the ones to check. To see a column's real name: open the list in SharePoint → gear icon → List settings → click the column → the Field= part of the URL is the internal name. In Power Apps formulas you use the display name, wrapped in single quotes if it contains spaces.

3 · Create the app

1Start a blank tablet app

Go to make.powerapps.com → make sure you are in the environment your client uses → Create → Blank app → Blank canvas app. Name it IBD Client Contact Repository and choose Tablet format → Create.

Power Apps Studio open on an empty screen called Screen1.

2Rename the screen

In the left Tree view, double-click Screen1 and rename it to scrMain.

scrMain at the top of the tree view.

3Set app settings

Settings (gear) → Display → turn Scale to fit off and Lock aspect ratio off, so the container layout resizes with the window. Close settings and save the app (Ctrl+S).

App saved; title bar shows IBD Client Contact Repository.

4 · Connect the data

1Add the SharePoint connection

Left rail → Data → Add data → search SharePoint → choose the SharePoint connector (sign in if prompted).

2Pick the site and the four lists

Paste the site URL https://bbkonline.sharepoint.com/sites/BMO → Connect → tick exactly these four lists → Connect:

  • Master List of Exposures
  • Borrower Contact Directory
  • Communication Purpose Master
  • Contact Communication Matrix
All four lists under Data in the left rail.
If a list is missing from the picker, it doesn't exist on this site under that name — go back to the Section 2 checklist before continuing.

5 · Build the layout

The whole screen is built from auto-layout containers, so it resizes cleanly. Structure: a vertical root → header bar on top → horizontal body → left (search) and right (detail) panels.

1Root container

With scrMain selected: Insert → Layout → Vertical container. Rename it conRoot and set:

conRoot — X / Y / Width / Height
X: 0        Y: 0
Width:  Parent.Width
Height: Parent.Height

In the right-hand panel set Gap to 0 and padding on all sides to 0.

2Header bar

Inside conRoot: Insert → Horizontal container, rename conHeader. Set Flexible height off and Height: 64. Set its Fill to the bank's red:

conHeader — Fill
RGBA(176, 17, 22, 1)

Inside it insert a Text label, rename lblAppTitle, set Text: "IBD Client Contact Repository", font size 18, bold, color white, vertical align center.

A red bar across the top with the app title in white.

3Body and the two panels

Inside conRoot (below the header): Insert → Horizontal container, rename conBody, leave Flexible height on so it fills the rest of the screen. Set Gap to 12 and padding 12 on all sides. Inside conBody insert two Vertical containers:

  • conLeft — Flexible width on, Fill portions 1, Fill RGBA(255,255,255,1), Gap 8, padding 12.
  • conRight — Flexible width on, Fill portions 2, Fill RGBA(255,255,255,1), Gap 12, padding 16.
A white ⅓-width panel on the left and ⅔-width panel on the right, on a light-grey background, red bar on top. Nothing in the panels yet.

4Save

Ctrl+S.

6 · Search panel

1Search box

Select conLeft → Insert → Text input (modern). Rename txtSearch. Set Placeholder (or HintText on classic): "Search by RIM or borrower name". Set Flexible height off.

2Borrower results gallery

Select conLeft → Insert → Vertical gallery (blank layout). Rename galBorrowers, turn Flexible height on so it fills the panel. Set its Items — this is the delegable search that stays fast at any list size:

galBorrowers — Items  (RIM stored as text verify)
Filter('Master List of Exposures',
    StartsWith(Title, txtSearch.Text)
    || StartsWith('Borrower RIM', txtSearch.Text))
galBorrowers — Items  (use this instead if RIM is a Number column)
Filter('Master List of Exposures',
    StartsWith(Title, txtSearch.Text)
    || (IsNumeric(txtSearch.Text) && 'Borrower RIM' = Value(txtSearch.Text)))

Inside the gallery template add two labels: borrower name (Text: ThisItem.Title, bold) and RIM (Text: "RIM: " & ThisItem.'Borrower RIM', smaller, grey).

Typing in the search box live-filters the borrower list.

3Selection loads the contact matrix

Set galBorrowers.OnSelect. This stores the selected borrower and pulls only that borrower's matrix rows into a local collection, enriched with the full contact record — one server call per selection, so the nested galleries in Section 8 never query SharePoint:

galBorrowers — OnSelect
Set(varBorrower, ThisItem);
Set(varLoading, true);
ClearCollect(colMatrix,
    AddColumns(
        Filter('Contact Communication Matrix', 'Borrower RIM'.Id = varBorrower.ID),
        ContactRec, LookUp('Borrower Contact Directory', ID = ThisRecord.Contact.Id)
    )
);
Set(varLoading, false)
If your Matrix stores RIM as plain text instead of a Lookup verify, replace the Filter condition with 'Borrower RIM' = varBorrower.'Borrower RIM'.

4Empty-search state

Insert a Text label into conLeft below the gallery, rename lblNoResults, Text: "No borrowers found", grey, centered.

lblNoResults — Visible
!IsBlank(txtSearch.Text) && CountRows(galBorrowers.AllItems) = 0
Typing gibberish shows "No borrowers found"; clearing the box hides it.
