IBD · Client Contact Repository · Build guide

One search, every contact

A view-only canvas app over one SharePoint list: type a RIM or borrower name, get the borrower's facility snapshot and everyone on the relationship — the BMO coverage team and the borrower's office contacts — straight from the Master List of Exposures. The app reads, never writes. Every step is self-contained, and every name and formula has a copy button.

What you're buildingSingle screen, master-detail: a search panel on the left, and on the right a borrower card (facility snapshot) plus two contact blocks — the BMO coverage team (RSMs, RM, RIM officer) and the borrower's office contacts (email addresses). Everything comes straight off the selected row of the Master List of Exposures. No Power Automate, no data entry, no other lists: the BMO team keeps maintaining the Master List exactly as they do today, and the app just makes it searchable.

Phase 1 — Foundation

A blank tablet app, one connection, and the container skeleton everything else sits in.

S1Create the app
  1. Go to make.powerapps.com → check you're in the client's environment (top-right) → Create → Blank app → Blank canvas app.
  2. Name: IBD Client Contact Repository · Format: Tablet → Create.
  3. Tree view → double-click Screen1 → rename to scrMain.
  4. Settings (gear) → Display → turn Scale to fit off and Lock aspect ratio off, so the layout resizes with the window. Save (Ctrl+S).
Check before moving onStudio open on an empty screen named scrMain, app saved under the right name.
S2Connect the Master List
  1. Left rail → Data → Add data → search SharePoint → pick the SharePoint connector (sign in if asked).
  2. Paste the site URL: https://bbkonline.sharepoint.com/sites/BMO → Connect.
  3. Tick exactly one list: Master List of Exposures → Connect.

Phase 2 — The search panel

Delegable search over the Master List, and the selection that powers everything on the right.

S4Search box & borrower list

a) The search box. Select conLeft → Insert → Text input → rename txtSearch. Flexible height off. Set the hint:

txtSearch · Placeholder (HintText on classic)

"Search by RIM or borrower name"

b) The results list. Select conLeft → Insert → Vertical gallery (Blank layout) → rename galBorrowers → Flexible height on. Set Items — delegable StartsWith, so it stays fast past 500/2000 rows:

galBorrowers · Items — if RIM is TEXT

Filter('Master List of Exposures', StartsWith(Borrower, txtSearch.Text) || StartsWith(RIM, txtSearch.Text))

galBorrowers · Items — if RIM is a NUMBER

Filter('Master List of Exposures', StartsWith(Borrower, txtSearch.Text) || (IsNumeric(txtSearch.Text) && RIM = Value(txtSearch.Text)))
If Power Apps flags BorrowerThe name column may still be the built-in Title column with its display name unchanged in Power Apps — the formula bar's IntelliSense will offer the right one as you type. Same fix everywhere Borrower appears.

c) The row template. Inside the gallery template add two labels — name on top, RIM under it:

Label 1 (bold) · Text

ThisItem.Borrower

Label 2 (smaller, grey) · Text

"RIM: " & ThisItem.RIM
Check before moving onIn Preview, typing narrows the borrower list as you type — by name and by RIM.
S5Selection powers the right panel

One line. Everything the right panel shows — facility figures, coverage team, office emails — is a column on the row the user just tapped, so remembering that row is the whole job.

galBorrowers · OnSelect

Set(varBorrower, ThisItem)
Why it's this simpleThe contact data lives on the Master List row — no second list to query, no collection to build, zero extra server calls per selection. The right panel just reads varBorrower.<column>.
Check before moving onSelect a borrower in Preview, then open the Variables pane: varBorrower holds that borrower's full record.
S6"No borrowers found"
  1. Select conLeft → Insert → Text label below the gallery → rename lblNoResults.
  2. Text "No borrowers found", grey, centered.
  3. Property dropdown → Visible → paste:

lblNoResults · Visible

!IsBlank(txtSearch.Text) && CountRows(galBorrowers.AllItems) = 0
Check before moving onGibberish in the search box shows the message; clearing the box hides it.

Phase 3 — The detail panel

Borrower card on top, then the heart of the app: the coverage team and the office contacts, straight off the selected row.

S7Placeholder & borrower card

a) Before any selection. Select conRight → Insert → Text label → rename lblPlaceholder → Text "Search for a borrower to see their contacts", grey, size 16, centered.

lblPlaceholder · Visible

IsBlank(varBorrower)

b) The card. Select conRight → Insert → Vertical container → rename conBorrowerCard. Flexible height off, Height 130, Fill RGBA(244, 245, 248, 1), padding 12.

conBorrowerCard · Visible

!IsBlank(varBorrower)

c) Its three labels. Everything reads from varBorrower — straight from the Master List, copied nowhere.

lblBorrowerName (size 20, bold) · Text

varBorrower.Borrower

lblBorrowerMeta (size 13, grey) · Text

"RIM: " & varBorrower.RIM & " · " & varBorrower.Office & " · " & varBorrower.'Facility Type' & " · Grade " & varBorrower.Grade

lblBorrowerFacility (size 14) · Text

"Limit " & Text(varBorrower.'Total Limit', "#,##0") & " · Exposure " & Text(varBorrower.'Current Exposure', "#,##0") & " · Utilization " & varBorrower.'Overall Utilization %' & "%"
Formatting the numbersText(..., "#,##0") assumes Number/Currency columns (what V1 expects). If a figure comes through as text, wrap it: Text(Value(varBorrower.'Total Limit'), "#,##0"). And if Utilization is stored as a fraction (0.85 not 85), use Text(varBorrower.'Overall Utilization %', "0%") instead.
Client wants more on the card?Any V1 "available extras" column drops in the same way — e.g. varBorrower.'Final Maturity' or varBorrower.'Syndicated or Bilateral'. Add a label, reference the column, done.
Check before moving onSelecting a borrower shows name, the RIM/office/grade line and the facility figures on the right; with nothing selected you see only the placeholder line.

Person vs lookup, in one sentenceA Person column carries the person record (.DisplayName, .Email); a Lookup carries its text through .Value and any extra fields as separate read-only columns named like 'BMO RSM 1: Email Address'. IntelliSense completes the right one as you type.
A blank name making an awkward row?Hide that row the same way — e.g. lblRM · Visible → !IsBlank(varBorrower) && !IsBlank(varBorrower.RM).
Check before moving onSelecting a borrower shows RSM 1, RSM 2, RM and RIM officer with their names from the list; tapping an RSM email opens a mail draft to that address.

c) The three email labels. One label each. The first two are extra fields riding on the Office (list) lookup; the third is the plain office email column (the lower-case name is real):

lblOfficeEmail1 · Text

lblOfficeEmail2 · Text

If a lookup-expansion name doesn't autocompletePower Apps names lookup extra fields exactly as SharePoint shows them in views — start typing varBorrower.'Office and take what IntelliSense offers; that's the authoritative spelling.
Still view-onlyLaunch("mailto:…") only opens the user's mail client with a pre-addressed draft — the app never writes to SharePoint and never sends anything itself.
Check before moving onThe office name and its emails show under the coverage block; a borrower with a blank 2nd email shows no empty row; tapping an email opens a draft.

Phase 4 — Finish & ship

S10Polish — clean state & selection highlight

a) Fresh start on every visit.

scrMain · OnVisible

Set(varBorrower, Blank())

b) Show which borrower is selected.

galBorrowers · TemplateFill

If(ThisItem.ID = varBorrower.ID, RGBA(31, 59, 110, 0.08), RGBA(0, 0, 0, 0))
Theme disciplineNavy RGBA(31,59,110,1) only on the header bar, the two block headings, email links and the highlight; body text RGBA(27,36,54,1); secondary grey RGBA(91,102,120,1). Segoe UI throughout (the default). Resist more color — it's a lookup tool.
S11Test everything

F5 from scrMain and run all ten. All must pass before publishing:

  • Open with nothing typed → full borrower list left, placeholder line right.
  • Partial borrower name → list narrows as you type.
  • Partial RIM → matching borrowers appear.
  • Gibberish → "No borrowers found".
  • Select a borrower → card shows name, RIM/office/grade line and the facility figures.
  • Coverage block shows RSM 1, RSM 2, RM, RIM officer names from the list.
  • Tap any email → a pre-addressed mail draft opens.
  • A borrower with a blank email (or blank 2nd office email) → that row is hidden, no empty gap.
  • Switch borrowers quickly → the right panel always matches the selection, no stale values.
  • App checker (stethoscope) → no delegation warnings on galBorrowers.
Check before moving onTen for ten. Any miss → the step that built that piece has the formula to re-check.
S12Publish & share — the victory lap
  1. File → Save → Publish → Publish this version.
  2. Power Apps home → the app's ⋯ menu → Share → add the IBD users or their security group as User (not Co-owner).
  3. Hand the BMO team this guide plus your filled-in V1 checklist, so future maintainers know the app's exact column dependencies.
Sharing the app does not grant list accessUsers also need Read on the Master List of Exposures — grant it on the BMO site. The app is view-only, so Read is all they need; don't grant Edit.
