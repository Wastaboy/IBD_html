Before you start

The two parts don't depend on each otherPart A (D1–D4) happens entirely in SharePoint and changes nothing in the app — no publish needed. Part B (U1–U6) happens entirely in Studio and touches only cosmetic properties — no controls added, no formulas edited, so nothing can break. Do them in either order.
How Part A works — the same trick the Matrix already uses, invertedThe Matrix's Borrower RIM column is a lookup: it shows the RIM and carries the name along as a read-only extra. Part A gives the Directory the mirror image: a lookup that shows the name (a searchable picker — type a few letters, pick) and carries the RIM along as the read-only extra. The RIM is derived from the picked row, so it can't be mistyped — and if a name or RIM ever changes on the Master List, the Directory follows automatically.
Chips: tap to copy — and the navy ones are controlsEvery inline chip is tap-to-copy (the ⧉ mark). A navy chip with a ▣ mark like conLeft is an element — a control in your Tree view; a plain grey chip is a name or value you paste. Property workflow, same as always: select the ▣ control in the Tree view → property dropdown (top-left of the formula bar) → paste the property name → paste the value → Enter.

Part A — Search by name in the Directory

Four steps in SharePoint. When they're done, nobody entering a contact ever types a RIM again.

D1Add the Borrower lookup column

Open the BMO site → Borrower Contact Directory. Then, the same lookup recipe as the Matrix build:

  1. Click + Add column (right end of the column headers) → pick Lookup → Next. You should now see the "Create a column" panel.
  2. Click into Name → paste Borrower.
  3. "Select a list as a source" dropdown → Master List of Exposures.
  4. "Select a column from the list above" dropdown → Borrower — that's the name column (the renamed Title).
  5. Expand More options → under "Add a column to show each of these additional fields" tick RIM.
  6. Click Save. You should now see TWO new column headers: Borrower and Borrower: RIM — the second is the read-only extra that fills itself. Two columns from one Save is correct, not a mistake (same as the Matrix's L3).
Can't find RIM in the additional-fields list?Only some column types can ride along on a lookup (text, number, date). RIM is a Number, so it is there — if it's missing, the source dropdown is pointing at the wrong list. Check step 3 before anything else.
Want the borrower to be mandatory?Edit the column later (column header → Column settings → Edit) → More options → toggle "Require that this column contains information". Leave it off until D2's backfill is done, or every old row will nag on every edit.
Check before moving onThe list shows the two new headers, both still empty on every existing row — D2 fills them.
D2Backfill the existing contacts
  1. Toolbar → Edit in grid view. You should now see the list turn into an editable grid.
  2. Click the first row's empty Borrower cell → type the first letters of that contact's borrower → the dropdown narrows as you type → pick. You should now see the Borrower: RIM cell fill by itself — it's read-only, don't try to type in it.
  3. Repeat for every contact → Exit grid view (same button).
A contact who serves more than one borrower?The Matrix still holds who-serves-whom-for-what — this column is just the Directory's own tag. If one person genuinely belongs to several borrowers, edit the column and toggle "Allow multiple selection"; the RIM column then lists each picked borrower's RIM.
Check before moving onEvery contact row shows a borrower name and a RIM — and not one of those numbers was typed by hand.
D3Put the new columns where the eye lands
  1. Drag the Borrower column header so it sits right after the contact's name (Title). If dragging fights you: column header → Column settings → Move left / Move right.
  2. Drag Borrower: RIM to sit right after Borrower.
Check before moving onReading a row left to right now tells the story in order: person → their borrower → its RIM → the contact details.
D4Verify the flow end to end
  1. Click + New. In the form, Borrower is a dropdown — type a few letters of any borrower → the list narrows → pick.
  2. Fill the contact fields as usual → Save.
  3. You should now see the new row with the borrower's name and its RIM — you never touched a number. (Delete the test row if it was a dummy.)
Nothing to publish — the app is untouchedThe app reads contacts through the Matrix and never looks at this column, so Part A needs no Studio work and no publish. (If the client later wants the borrower shown on the contact cards in the app, it's one label away — say the word.)
Check before moving onOpen the app once and search a borrower — everything behaves exactly as before. Part A done.

Part B — The polish pass

Six steps in Studio. Cosmetic properties only — the row math, formulas and control tree stay exactly as they are.

What this pass deliberately does NOT touchNo text sizes inside the gallery rows — those sit in carefully balanced fixed-anchor math (TemplateSize, Y offsets), and bumping them would cascade. The wins below are all free: corners, shadows, spacing, casing, one font.

U5One font, everywhere, on purpose

Depending on which control flavor each insert used, a canvas app can quietly mix Open Sans and Segoe UI — close enough to miss, different enough to look "off". Ten seconds of multi-select makes the font deliberate:

  1. In the Tree view, click lblAppTitle, then hold Ctrl and click every other label in the app — including the ones nested inside the galleries and containers — plus txtSearch.
  2. With them all selected, property dropdown → Font → paste:
Font.'Segoe UI'
Prefer a touch more character?Font.Lato is the one tasteful alternative on the canvas palette — set it the same way. Whichever you pick: one font for the whole app, never a mix.
Check before moving onSweep the app in Preview: every piece of text — title, rows, card, headings, emails — renders in the same face.
U6Save, publish, sweep

Part B changed the app, so the double-click from the final-build page applies:

  1. Save (Ctrl+S) → "All changes saved".
  2. Publish → Publish this version.
  3. Open the app from the homepage button and do a thirty-second sweep: search a borrower, select, hover the list, tap an email. Nothing structural changed — this is confirmation, not testing.
The finish lineRounded cards on a calm grey, an aligned confident title, quiet section labels, one typeface — and in SharePoint, contacts entered by name with RIMs that write themselves. 🏁
