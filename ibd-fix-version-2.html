The nine steps

F1SharePoint: remake Active Status as Yes/No

SharePoint can't convert a Choice column to Yes/No, so delete and recreate — painless because only the two test contacts exist:

  1. Open Borrower Contact Directory in SharePoint. You should now see the list with your two test contacts.
  2. Click the Active Status column header → Column settings → Edit. You should now see the edit panel slide in from the right.
  3. Scroll to the bottom of the panel → click Delete → confirm. You should now see the column disappear.
  4. + Add column → Yes/No. You should now see the "Create a column" panel.
  5. Name → paste Active Status → Default value dropdown → Yes → Save.
  6. Fix the two test rows: Test Contact One → Active Status Yes · Test Contact Two → No (edit each row: hover → ✏️ or open the item → change → Save).
Why Yes/No mattersThe contacts filter is ContactRec.'Active Status' — a clean true/false test. Against a Choice column that's a type error, and "fixing" it by comparing strings would leave a typo-shaped hole ("active" vs "Active"). The column type is the contract.
Check before moving onThe Directory shows Active Status as Yes/No toggles; Contact One = Yes, Contact Two = No.
F2Studio: refresh the fixed list & clear the wreckage
  1. Open the app in Studio → left rail → Data. Hover Borrower Contact Directory → ⋯ → Refresh, so Power Apps picks up the remade Active Status column.
  2. Hover CustomGallerySample → ⋯ → Remove. (It sneaked back in when a gallery was inserted with a sample layout — F4 explains how to avoid that.)
  3. In the Tree view, right-click galContacts (the one sitting directly under conRight, with Label8 inside) → Delete. You should now see it and Label8 gone. It has to be rebuilt inside galPurposes — a gallery can't be moved into another gallery's template, only created there.
Check before moving onData shows exactly four sources; the Tree view under conRight has no galContacts and no Label8.
F3Paste the corrected galPurposes formula

Your Items formula is the guide's old version, which had a genuine bug (a bare Title inside the inner Filter gets captured by the Matrix rows' own blank Title — every purpose counts zero, so no headings ever render). Select galPurposes → property dropdown → Items → select everything → paste:

galPurposes · Items — the As PurposeRow alias is the fix

Filter('Communication Purpose Master' As PurposeRow, CountRows(Filter(colMatrix, Purpose.Value = PurposeRow.Title)) > 0)
Check before moving onNo red underline on the formula. (Headings still won't show — the template is empty until F4.)
