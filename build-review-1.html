Setting any Power Apps property — the same three pastes, always
  1. Select the control in the Tree view (left rail — its name is always a chip on this page).
  2. Pick the property: click the property dropdown at the top-left of the formula bar and type the property chip, e.g. LayoutAlignItems.
  3. Paste the value into the formula bar, e.g. LayoutAlignItems.Stretch. Enter. Next row.

Two translations, same as the main guide: the right-panel's Flexible height/width toggle is the FillPortions property (0 = off / fixed size · 1 = on / fill the space), and the panel's Align (horizontal) is LayoutAlignItems.

LayoutAlignItems.Stretch
Same bug as round oneCenter makes every child shrink to its minimum width and float in the middle — the card, coverage and office blocks end up in a skinny centered column. Stretch makes them fill the panel edge-to-edge. If a layout ever looks pinched, this property is always the first suspect.

F3Install the real selection formula

Your galBorrowers OnSelect still holds the old one-liner. This paste fills colMatrix with the selected borrower's matrix rows, each enriched with the full contact record — it's what makes your two app-checker errors and the "unused varLoading" warning disappear:

  1. In the Tree view, click galBorrowers (inside conLeft). You should now see the gallery selected on the canvas.
  2. Click the property dropdown (top-left of the formula bar) → type OnSelect → select it. You should now see the formula bar showing Set(varBorrower, ThisItem).
  3. Click into the formula bar → select everything (Ctrl+A) → paste:

galBorrowers · OnSelect — replace everything

Set(varBorrower, ThisItem); Set(varLoading, true); ClearCollect(colMatrix, AddColumns( Filter('Contact Communication Matrix', 'Borrower RIM'.Id = varBorrower.ID), ContactRec, LookUp('Borrower Contact Directory', ID = ThisRecord.Contact.Id) ) ); Set(varLoading, false)
Red squiggles until F1 is doneThis formula names the two lists F1 connects — paste it after F1 or it can't resolve them.
Check before moving onSelect your test borrower in Preview → Variables pane: colMatrix holds that borrower's matrix rows with a ContactRec record in each. App checker (stethoscope): the two scrMain.OnVisible errors are gone.
